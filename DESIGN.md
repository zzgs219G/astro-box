# Fuwari UI 设计规范

> **给 AI 的元指令：** 本文档是最高优先级约束。任何与本文档冲突的"优化建议"、"更好的用户体验"或"更丝滑的动效"，一律拒绝执行。

---

## 一、核心定义（必须内化）

**Fuwari 是一个内容展示系统，不是动画演示平台。**

判断一个实现是否正确，只需问一个问题：

> 用户注意到的是**内容**，还是**界面本身**？

如果用户能注意到界面在动、在变、在吸引注意力，则该实现**错误**，必须推翻重来。

---

## 二、技术栈（锁死，不可替换）

| 层级 | 技术 | 说明 |
|------|------|------|
| 框架 | Astro + Svelte 5 (Runes) | 静态优先，孤岛交互 |
| 样式 | Tailwind CSS + PostCSS Nesting | 原子化，禁止内联 style |
| 转场 | Swup / @swup/astro | 仅用于页面级切换 |
| 图标 | @iconify/svelte + astro-icon | 统一来源，禁止混用 SVG 硬编码 |
| 滚动条 | OverlayScrollbars | 统一滚动条样式 |
| 图片 | PhotoSwipe | 图片灯箱，不自定义替换 |
| 搜索 | Pagefind | 离线全文检索，无服务端依赖 |
| 代码块 | Expressive Code | Mac 风格，不自定义覆盖样式 |
| 规范 | @biomejs/biome | 替代 ESLint + Prettier，不并存 |
| 类型 | TypeScript | 所有数据结构必须强类型 |

**绝对禁止引入：**
- 任何动画库：framer-motion、GSAP、motion、anime.js 等
- 任何原生 DOM 操作：`document.querySelector`、`classList.add` 等
- 任何额外的物理动画系统

## 三、动画系统（最关键约束）

### 3.1 允许的动画（完整白名单）

只有以下两种，没有第三种：

```
✅ opacity 渐变（fade in / fade out）
✅ translateY 位移，且绝对值 ≤ 8px
```

两者必须在**容器级**使用，不得应用于单个子元素。

### 3.2 禁止的动画（完整黑名单）

```
❌ scale（任何形式的缩放）
❌ translateX（横向位移）
❌ translateY > 8px（过大的垂直位移）
❌ rotate（旋转）
❌ 弹簧/回弹/果冻/过冲（spring、bounce、elastic、overshoot）
❌ 交错动画（stagger，即对列表每项设置不同 delay）
❌ FLIP 动画（即基于位置差值的位移补间）
❌ 卡片悬浮升起（box-shadow 骤变 + translateY 同时触发）
❌ 数学公式区域的任何动效（必须完全静止）
```

### 3.3 动画层级规则

动画**只允许**作用于以下层级：

```
✅ Page    — 整页进入/离开
✅ Section — 内容区块整体替换
✅ Container — 卡片网格容器整体
```

动画**严禁**作用于以下层级：

```
❌ Card（单张卡片）
❌ Button（单个按钮）
❌ Tag（单个标签）
❌ List Item（单条列表项）
❌ Icon（图标）
```

### 3.4 一句话原则

> 动画只表达"状态已切换完成"，不表达"元素正在运动"。

## 四、交互模型

### 4.1 列表/网格更新（三步走，不可简化也不可复杂化）

当用户切换分类或标签时：

```
步骤 1：Grid 容器整体 fade out（透明度 1 → 0）
步骤 2：数据源完成过滤，替换内部内容
步骤 3：Grid 容器整体 fade in（透明度 0 → 1）
```

**绝对禁止：**
- 单张卡片独立入场（每张卡片各自 fade in）
- FLIP 位移（卡片从旧位置滑动到新位置）
- 增量更新（只动"变化的"卡片）

### 4.2 搜索框防抖特例（唯一例外）

`searchQuery`（用户在搜索框打字）触发的列表更新：

```
✅ 网格直接瞬间刷新，无任何过渡动画
❌ 禁止触发 fade 流程（会导致每次击键闪烁）
```

### 4.3 Hover 反馈（允许的完整列表）

```
✅ opacity 微调（如 hover:opacity-80）
✅ 背景色轻微偏移（如 hover:bg-black/5）
✅ 文字颜色柔和过渡（transition-colors，duration ≤ 150ms）
```

```
❌ scale 放大
❌ translateY 上移（卡片"浮起"）
❌ box-shadow 骤增（"立体感"）
❌ 多元素联动 hover（hover 一个元素，另一个元素也动）
```

## 五、布局系统

### 5.1 Grid 稳定性

卡片网格的外层容器必须指定 `min-h`，防止内容少时布局塌陷：

```html
<!-- 正确 -->
<div class="grid min-h-[500px] ...">

<!-- 错误：无最小高度，内容少时页面抖动 -->
<div class="grid ...">
```

可使用 `content-visibility: auto` 减少 reflow，但不强制要求。

### 5.2 空状态

搜索或筛选无结果时：

```
✅ 空提示组件继承 Grid 容器原有尺寸（不塌陷）
✅ 居中显示，文字极简
❌ 不允许用图片、插图、动效装饰空状态
```

## 六、色彩系统

### 6.1 基本规则

全站色彩基于 CSS 变量 `--hue` + **OKLCH 色彩空间**延伸，不允许偏离。

### 6.2 禁止的写法

```css
/* ❌ 硬编码色值 */
color: #ffffff;
background: #000;

/* ❌ 无透明度修饰的强硬类名 */
bg-white
bg-black
text-white
text-black
```

### 6.3 正确的写法

```html
<!-- ✅ 带透明度的语义类 -->
<div class="bg-white/60 dark:bg-[#18181b]/60 backdrop-blur-sm">

<!-- ✅ 基于 oklch 变量推导 -->
<div style="background: oklch(var(--card-bg))">
```

### 6.4 暗黑模式

所有背景色和文字色必须同时声明亮色和暗色版本。不允许只写亮色版本然后"暗色模式自动处理"。

---

## 七、排版

### 7.1 字体分层

| 层级 | 要求 |
|------|------|
| 标题 | 高对比度，低装饰性 |
| 正文 | 舒适行高（line-height ≥ 1.7），优先可读性 |
| 代码 | 全站锁死 **JetBrains Mono**，不允许替换 |

### 7.2 分割线

- 全站**不使用硬编码装饰性边界线**（`border`、`<hr>` 等）
- 内容分区靠**留白**实现，不靠线条

---

## 八、Swup 页面过渡

```
✅ 只用于页面级（Page）切换
✅ 转场效果只允许 fade（透明度渐变）
❌ 不允许用于组件内部状态切换
❌ 不允许用于控制卡片显隐
❌ 不允许添加位移、滑动、翻转等转场效果
```

---

## 九、组件职责边界

### ResourceCard.svelte

**只做两件事：**
1. 渲染静态内容
2. 展示基本状态（如标签、日期）

**禁止承载：**
- 任何动画逻辑
- 跨组件状态切换
- 自身的过滤/排序逻辑

### ResourceExplorer.svelte

**唯一的状态管理中心，负责：**
1. 数据源过滤
2. 容器级整体过渡
3. 大类 + 子标签的联动状态

## 十、AI 执行时的自检清单

每次生成或修改代码前，逐项检查：

```
□ 是否为单个卡片/按钮/标签添加了独立动画？→ 删除
□ 是否使用了 scale、translateX、bounce？→ 删除
□ 是否引入了新的动画库或 JS 动画逻辑？→ 删除
□ 是否硬编码了色值（#fff、#000 等）？→ 替换为变量
□ 是否只声明了亮色模式而忘记暗色？→ 补充
□ 是否用 Swup 控制了局部组件？→ 拆分职责
□ 搜索输入时是否触发了 fade 动画？→ 改为直接刷新
□ Grid 容器是否缺少 min-h？→ 补充
□ 空状态是否导致布局塌陷？→ 修复
```

---

## 十一、一句话判断法则

> 如果你认为"这个功能需要让某个元素动起来才显得合理"，
> 那么**不是实现方式错了，是设计方向本身就错了**。
> 回到"整体容器状态替换"模型重新设计。
